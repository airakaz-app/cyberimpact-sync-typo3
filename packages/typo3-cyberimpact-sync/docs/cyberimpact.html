    Cyberimpact API

    Welcome to the Cyberimpact API. With it, you can interact with your Cyberimpact account programmatically. Our API is based on REST principles. The following documentation covers core resources.

    First of all, read these basic instructions to start using the API:

    • You must authenticate to the Cyberimpact’s API with a JSON Web Token (JWT) generated in the application (or via the API) using the Authorization: Bearer token-content HTTP header

    • The URL for the Cyberimpact API is located at https://api.cyberimpact.com

    • Submit all your requests (for POST, PUT, PATCH) in JSON

    • Submit all your requests using the UTF-8 character encoding

    • Cyberimpact API uses standard HTTP response codes to indicate the success or failure of a request. Codes in the 2xx range indicate success. Codes in the 4xx range indicate an error.

    Authentication ¶

    The Cyberimpact API uses API tokens to authenticate requests. You can view and manage your API tokens in the Developers / API tokens menu of the Cyberimpact application.

    Your API tokens carry many privileges, so share them securely and only with people you trust. They are the access to your account, so this information should not be public.

    Authentication to the API is performed via bearer authentication. Just add a "Authorization: Bearer YOUR_JWT_ACCESS_TOKEN" HTTP header to each of your requests. You can browse the Resources section of this documentation for examples in different programming languages.

    Universal parameters ¶

    dateReturnFormat ¶

    The Cyberimpact API will by default return every date in the format Y-m-d H:i:s.u, in the EST/EDT timezone.

    If you need a standard date format, you can add the dateReturnFormat parameter in the query string to have any query return a standard date format with timezone.

    Available values:

    • ISO8601

    • ATOM

    Example:

    https://api.cyberimpact.com/members?dateReturnFormat=ISO8601

    This parameter will not allow you to send these date formats, you will still have to send dates using the format shown in this documentation.

    Retrieve members
    GET/members{?page,limit,status,sort,joinedOnFrom,joinedOnTo,updatedOnFrom,updatedOnTo}

    Retrieve a paginated list of your members.

    Active members are subscribed members that are part of group(s).

    Orphan members are subscribed members that are not part of a group.

    The returned property items_per_page is deprecated and will eventually be removed. Use limit instead.

    Example URI

    GET https://api.cyberimpact.com//members?page=&limit=&status=&sort=&joinedOnFrom=&joinedOnTo=&updatedOnFrom=&updatedOnTo=

    Add a batch
    POST/batches

    Batches submitted to Cyberimpact API can be of two types, addMembers, unsubscribe and deleteMembers. Please refer to their respective schema section to learn how to format your batches for processing. If you are not familiar with JSON schema, you can learn more here. You can also consult our troubleshooting guide here for more details and an example using a JSON Schema validator.

    Details for addMembers batch :

    • If a member already exists, it is updated with the new information provided in the batch.

    • If a field is empty in the batch and there is already data present for that field for a member in your account, we keep the data you already have for that member.

    • If you specifically want to empty a field, provide the special string __EMPTY__ in your batch to clear that field.

    • All submitted members will be added to the groups specified in the batch (if they are not already in). If they were already in other groups, they will stay in their previous groups in addition to being added to the new ones.

    • The available values for the relationType field are :

      • express-consent
      • active-clients
      • information-request
      • business-card
      • web-contacts
      • purchased-list
      • contest-participants
      • mixed-list
      • inactive-clients
      • association-members
      • employees
      • partners
